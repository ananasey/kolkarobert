# Streamlit-проект: городская среда и оптимальный маршрут

Учебное Streamlit-приложение объединяет два задания:

1. Автоматически получает спутниковый снимок Яндекс.Карт для выбранного города и анализирует качество городской среды.
2. Геокодирует 7 населённых пунктов РФ, рассчитывает дорожные расстояния, стоимость доставки и строит оптимальный замкнутый маршрут одной машины.

Проект сделан как демонстрационный, но рабочий стенд: код разделён на модули, API-ключи не хранятся в коде, ошибки обрабатываются в интерфейсе, а для недоступного дорожного маршрута предусмотрен честный fallback.

## Установка на macOS

Откройте терминал в папке проекта и выполните:

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

Если `python3.11` не найден, установите Python 3.11+ через официальный установщик Python или Homebrew.

## Настройка .env

Создайте файл `.env` рядом с `app.py`:

```bash
cp .env.example .env
```

Заполните ключи:

```env
# Необязательный общий ключ. Заполняйте его только если один ключ
# действительно имеет доступ ко всем нужным API.
YANDEX_API_KEY=

# Рекомендуемый вариант: отдельные ключи под отдельные API.
YANDEX_GEOCODER_API_KEY=ключ_для_геокодера
YANDEX_STATIC_MAPS_API_KEY=
YANDEX_DISTANCE_MATRIX_API_KEY=ключ_для_distance_matrix
```

Если кабинет Яндекса разрешает выбрать только один API при подключении, создайте
или подключите три ключа и вставьте каждый в свою строку. Если у вас уже есть
один общий ключ, приложение тоже его поддерживает через `YANDEX_API_KEY`.

Ключи должны иметь доступ к нужным сервисам Яндекса:

- Geocoder API
- Distance Matrix API

Для спутникового снимка используется Static Maps `1.x` со слоем `l=sat`. В текущем варианте Яндекс отдаёт этот спутниковый слой без передачи ключа в запросе, поэтому `YANDEX_STATIC_MAPS_API_KEY` можно оставить пустым. Остальные ключи всё равно хранятся только в `.env`.

## Как получить ключ Яндекса

1. Откройте кабинет разработчика Yandex Maps API: https://developer.tech.yandex.com/
2. Создайте приложение или ключ API.
3. Если можно выбрать только один API, подключите ключи по очереди:
   - `JavaScript API и HTTP Геокодер` -> `YANDEX_GEOCODER_API_KEY`;
   - `Матрица Расстояний и Построение Маршрута` -> `YANDEX_DISTANCE_MATRIX_API_KEY`.
4. Скопируйте ключи в `.env`.

Точные названия услуг и условия тарификации могут меняться, поэтому перед демонстрацией стоит проверить, что все три API активны в кабинете разработчика.

## Задание 1: анализ городской среды

Вкладка `Анализ городской среды` работает так:

1. Пользователь вводит только название города.
2. Приложение получает координаты центра через Yandex Geocoder API.
3. Приложение загружает спутниковый снимок через Yandex Static Maps API со слоем `l=sat`.
4. Пользователь выбирает один из трёх видов анализа.
5. Приложение показывает исходный спутниковый снимок, маску, итоговую визуализацию, численную метрику и текстовый вывод.

Анализ выполняется по спутниковому изображению. Это лучше схемы карты, потому что на снимке видны реальные кроны деревьев, крыши, дворы, дороги, промышленные площадки и открытые территории. Схема карты удобна для навигации, но её цвета являются условным дизайном, а не фактическим видом городской среды.

### Зелёные зоны

Модуль `analysis/green_analysis.py` использует OpenCV:

1. изображение переводится из RGB в HSV;
2. выделяется диапазон зелёных оттенков;
3. маска очищается морфологическими операциями `open` и `close`;
4. считается процент зелёных пикселей и примерная площадь зелёных зон.

Диапазон HSV в OpenCV:

```text
H: 35-90
S: 35-255
V: 35-255
```

Дополнительно используется проверка баланса RGB, чтобы случайные цветные шумы реже попадали в зелёную маску.

Результат:

- процент озеленения;
- примерная площадь зелёных зон в км²;
- чёрно-белая маска;
- наложение маски на спутниковый снимок;
- текстовая оценка: низкое, среднее или высокое озеленение.

Площадь считается приближённо: приложение оценивает метров на пиксель по широте центра и zoom Web Mercator, затем умножает количество зелёных пикселей на площадь одного пикселя.

### Плотность и тип застройки

Модуль `analysis/building_analysis.py` оценивает застройку по признакам спутникового снимка:

1. изображение переводится в HSV и grayscale;
2. исключаются зелёные и водные участки;
3. выделяются светлые/серые поверхности с низкой насыщенностью, похожие на крыши, асфальт и плотные городские покрытия;
4. применяется CLAHE для усиления локального контраста;
5. применяется Canny edge detection;
6. рассчитывается плотность границ, текстурность и количество значимых контуров.

Результат:

- индекс плотности;
- процент светло-серых/текстурных зон;
- плотность границ;
- количество значимых контуров;
- классификация: низкая, средняя или высокая;
- маска и итоговая визуализация.

### Транспортная инфраструктура

Модуль `analysis/transport_analysis.py` ищет дорожную сеть как светлые/серые линейные структуры:

1. исключаются зелёные и водные области;
2. выделяются светлые поверхности с низкой насыщенностью;
3. применяется Canny edge detection;
4. морфология соединяет близкие фрагменты;
5. через контуры ищутся длинные вытянутые структуры.

Результат:

- доля дорожной сети;
- количество длинных линий;
- индекс транспортной развитости;
- маска и итоговая визуализация;
- краткая интерпретация.

## Ограничения спутникового CV-анализа

Это учебный анализ, а не промышленная GIS-классификация. Он реалистичнее анализа схемы карты, но всё равно зависит от:

- сезона и времени съёмки;
- теней от зданий и деревьев;
- облаков и качества спутникового снимка;
- масштаба карты;
- похожести крыш, парковок, дорог и открытого грунта;
- того, что на спутнике нет инфракрасного канала, поэтому NDVI посчитать нельзя;
- плотности объектов на выбранном фрагменте.

Преимущество такого подхода для учебного проекта: логика прозрачна, использует реальные методы компьютерного зрения, код можно объяснить на защите, результат воспроизводим, а снимки загружаются автоматически.

## Задание 2: оптимальный маршрут

Вкладка `Оптимальный маршрут` работает так:

1. Пользователь вводит ровно 7 населённых пунктов РФ, каждый с новой строки.
2. Приложение проверяет количество пунктов и дубли.
3. Каждый пункт геокодируется через Yandex Geocoder API.
4. Пользователь выбирает стартовый город.
5. Пользователь задаёт тариф за 1 км.
6. Приложение запрашивает дорожную матрицу через Yandex Distance Matrix API.
7. Строится матрица расстояний и матрица стоимости.
8. Точная задача коммивояжёра решается полным перебором перестановок.
9. Маршрут замыкается в стартовый город.

Формула стоимости:

```text
cost = distance * tariff
```

Для 7 пунктов полный перебор подходит хорошо: фиксированный старт оставляет `6! = 720` вариантов, поэтому решение получается точным и быстрым.

## Дорожные расстояния и fallback

Основной режим использует Yandex Distance Matrix API с параметром `mode=driving`.

Если дорожная матрица недоступна, приложение не скрывает проблему и не выдаёт прямое расстояние за дорожное. Оно явно показывает предупреждение и переключается на fallback:

- рассчитывает геодезические расстояния между координатами по формуле haversine;
- помечает режим как `геодезия`;
- продолжает строить маршрут и стоимость уже по fallback-матрице.

Причины fallback:

- не задан API-ключ;
- Distance Matrix API не подключён к ключу;
- HTTP-ошибка;
- превышены лимиты API;
- API не смог построить маршрут для одной или нескольких пар.

## Обработка ошибок

В приложении обработаны ситуации:

- отсутствует общий `YANDEX_API_KEY` или отдельный ключ нужного сервиса;
- HTTP-ошибка API;
- API вернул пустой или неожиданный ответ;
- город не найден;
- геокодер вернул несколько вариантов;
- введено меньше или больше 7 пунктов;
- есть дублирующиеся пункты;
- стартовый город не выбран;
- тариф некорректен;
- дорожное расстояние недоступно.

Ошибки показываются пользователю через `st.error`, `st.warning` или `st.info`, приложение при этом не падает.

## Кеширование

Кеш хранится в `data/cache`.

Кешируются:

- результаты геокодирования;
- изображения Static Maps;
- матрицы расстояний.

Это ускоряет повторные демонстрации и снижает количество запросов к API.

## Что показать преподавателю

Во время демонстрации удобно показать:

1. Ввод города и автоматическую загрузку спутникового снимка.
2. Три анализа одного фрагмента: зелёные зоны, застройка, транспорт.
3. Маски и численные метрики.
4. Ввод 7 населённых пунктов.
5. Таблицу координат после геокодирования.
6. Матрицу расстояний и матрицу стоимости.
7. Сообщение о режиме расстояний: дорожный API или fallback.
8. Итоговый маршрут, общую длину, стоимость и граф маршрута.

Проект защищаем тем, что он не обещает идеальную научную точность анализа изображения, но честно реализует автоматическое получение данных, объяснимые CV-методы, дорожную маршрутизацию с fallback и точное решение TSP для заданного размера.
